What makes a good first project?

  • Small scope (2 hours max)
  • Something you actually want
  • Simple is better
  • A landing page, a tool, a game

You can always add more later. Ship first.

Idea Generation

Ask yourself:

What wasn't possible before because of constraints, but is possible now?
  • That side project you never had time for
  • That tool you wished existed
  • That thing you'd hire a developer to build

Tips for Claude Code

  • Be specific about what you want
  • Give context (what you're building, why)
  • Paste errors directly back to Claude
  • Ask it to explain if you're confused
  • Start simple, then iterate

When You're Stuck

The Loop:

1. Try something

2. Read the error

3. Paste to Claude

4. Repeat

This is 90% of the work.
